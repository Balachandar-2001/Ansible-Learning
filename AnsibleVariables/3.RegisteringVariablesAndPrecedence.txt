WHAT IF WE HAVE A VARIABLE DEFINED ON THE TWO PLACES?

WHILE DECLARING THE HOSTS ON THE host file,

We can define the variable on the host level or the group level.

Example:
    /etc/ansible/hosts:
        web1 ansible_host-172.20.1.100
        web2 ansible_host-172.20.1.100  dns_server=10.5.5.4 // host variable for the web2 server priority 1
        web3 ansible_host-172.20.1.100

        [web_servers]
        web1
        web2
        web3
        
        [web_servers:vars]
        dns_server=10.5.5.3 // host group variable priority 2

    While running the ansible, variables stored on [web_servers:vars] would be distributed to web1, web2 and web3 but host variable has been defined for web2 web2 would use that host variable!

    If the variable is defined on the play level [ defined on the playbook ], it would be having more priority than the hosty level variable

    If any variables has been passed during the execution of the ansible on the command line as the extra vars,
        Example:
            ansible-playbook playbook.yaml --extra-vars "dns_server=10.5.5.6"
        In this case this would take the highest presedence.

    Precedence low to high:

    Group vars                          |
    Host vars                           |
    Palybook vars                       |    
    Extra Vars during cmd line run.     |
                                       \ /
                                        |


To know more about the precedence go to the official ansible doc on variables. there are 22 ways of defining the variables and each having different precedence 

Lowest - Role Default variables
Highest- Extra Vars.

HOW TO CAPTURE THE O/P OF THE FIRST COMMAND AND PASS IT TO THE NEXT COMMAND?

On the Playbook,
    tasks:
        - shell: cat /etc/hosts
          register: result // set the result
        - debug:
            var: result   // putting the result on the result variable.
